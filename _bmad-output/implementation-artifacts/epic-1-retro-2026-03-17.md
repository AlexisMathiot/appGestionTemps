## Succès & Points forts

1. **Design épuré et simple** — Le choix DaisyUI + Tailwind CDN a produit un design propre qui plaît, sans complexité de build. Base solide pour les fonctionnalités à venir.
2. **Flux auth complet et fonctionnel** — Inscription, connexion, déconnexion, reset password livrés et fonctionnels sans bugs remontés.
3. **Progression fluide entre stories** — Patterns HTMX établis en Story 1.2-1.3 réutilisés efficacement en 1.4-1.5. Helpers factorisés (`_set_session_cookie`, `_redirect`) ont accéléré les stories suivantes.
4. **Suite de tests solide** — Croissance organique de 9 → 23 → 33 → 48 → 59 tests. Couverture routes, modèles, fonctionnel auth.
5. **Capitalisation des apprentissages** — "Previous Story Intelligence" et "Debug Log References" dans chaque story ont évité de retomber dans les mêmes pièges.
6. **Choix techniques audacieux payants** — Python 3.14, PostgreSQL 18, versions récentes : approprié dans le contexte projet personnel.

## Difficultés & Défis

1. **Incompatibilités versions récentes (3/5 stories impactées)**
   - Story 1.1 : PostgreSQL 18 volume mount différent, pytest-asyncio mode STRICT
   - Story 1.2 : TemplateResponse API changée dans Starlette récent
   - Story 1.3 : passlib incompatible bcrypt 5.0+ sous Python 3.14, datetime.utcnow() deprecated, conftest.py NullPool nécessaire pour asyncpg
   - **Consensus équipe :** Prix acceptable pour tester des outils récents sur un projet personnel. Python 3.14 apporte des gains de performance et le support des template strings.

2. **CSRF protection reportée** — Mentionnée dans 3 stories comme "à faire dans la prochaine", jamais implémentée. Décision consciente : non-bloquant tant que l'app reste en local.

## Enseignements clés

1. Les versions récentes apportent de la valeur mais nécessitent de **vérifier la compatibilité des librairies en amont**
2. La **factorisation progressive** entre stories (helpers, conftest, patterns HTMX) accélère significativement le développement
3. La **capitalisation systématique** (Debug Logs, Previous Story Intelligence) évite de retomber dans les mêmes pièges
4. Un **design épuré dès le départ** crée une base solide pour les fonctionnalités à venir
